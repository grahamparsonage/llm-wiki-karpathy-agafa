## Key points
- Semantic Web reasoning combines metadata with machine-processable inference methods.
- Rules and ontologies are treated as complementary formalisms, but integration raises semantic and computational trade-offs.
- RDF/RDFS/OWL semantics and query languages (for example SPARQL-family approaches) are central to practical reasoning workflows.
- Inference patterns (including closure and rule-based derivations) are needed to make implicit knowledge explicit.
- Language design and evaluation strategy significantly affect expressiveness, performance, and usability.
