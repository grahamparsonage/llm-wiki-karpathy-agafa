Foundational overview of answer set programming (ASP) explaining stable model semantics, solver mechanisms, core modeling patterns, and practical applications.

## Citation
Lifschitz, V. (2008). *What Is Answer Set Programming?* In Proceedings of the Twenty-Third AAAI Conference on Artificial Intelligence.

## Key points
- ASP is a declarative programming paradigm for difficult search problems, especially knowledge-intensive tasks.
- Computation is based on stable model semantics rather than Prolog-style SLDNF execution.
- Modern ASP solvers share search ideas with SAT solving and are designed to terminate on grounded programs.
- Common modeling pattern is **generate, define, test** (choice rules, auxiliary definitions, constraints).
- ASP supports both negation as failure and strong/classical negation for representing incomplete knowledge.
- Stable model semantics gives a formal basis for deriving implicit conclusions from rule sets.
